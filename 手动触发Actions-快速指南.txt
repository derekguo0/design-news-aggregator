⚡ 最快方法（4步走）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第1步：访问 Actions 页面
────────────────────────────────────────────────────
URL格式：
https://github.com/[您的用户名]/[仓库名]/actions

或从仓库页面点击顶部 "Actions" 标签

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第2步：选择 Workflow
────────────────────────────────────────────────────
在左侧列表中，点击：
"Auto Deploy to Vercel"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第3步：运行 Workflow
────────────────────────────────────────────────────
1. 找到右上角蓝色按钮 "Run workflow"
2. 点击按钮
3. 弹出窗口中，确认分支为 "main"
4. 点击绿色 "Run workflow" 按钮

┌─────────────────────────────────┐
│ Run workflow                    │
│ Branch: main              ▼     │
│      [Run workflow]             │ ← 点这个
└─────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第4步：查看结果
────────────────────────────────────────────────────
触发后，页面会显示新的运行记录：

🟡 正在运行... (黄色圆圈)
   等待 2-3 分钟

✅ 运行成功！(绿色对勾)
   所有步骤都是绿色 ✅

❌ 运行失败 (红色叉号)
   点击查看错误日志

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3. 没有红色错误

❌ 如果失败了
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

点击失败的运行记录 → 查看红色❌步骤 → 展开查看错误

常见错误：

错误1: "Bad credentials" 或 "Resource not accessible"
解决: 检查 VERCEL_TOKEN 是否正确，重新创建 Token

错误2: "Project not found"
解决: 检查 PROJECT_ID 和 ORG_ID 是否正确

错误3: 某个步骤失败
解决: 点击步骤查看详细日志，根据错误信息处理
