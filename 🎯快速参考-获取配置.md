# 🎯 快速参考：获取Vercel配置

## 一、获取Vercel Token（2分钟）

### 📍 访问路径
```
https://vercel.com → 头像 → Settings → Tokens → Create Token
```

### 📝 填写信息
- **名称**: `GitHub Actions Deploy`
- **过期**: `No Expiration`（永不过期）
- **作用域**: 选择您的账户/团队

### ⚠️ 重要
**Token只显示一次！立即复制保存！**

格式示例：`kQpX9YmZnAbCdEfGhIjKlMnOpQrStUvW`

### 方法B：Vercel控制台

```
https://vercel.com → 选择项目 → Settings → General
```

找到：
- **Project ID**: `prj_xxxxxxxxxxxxxxxxxxxx`
- **Organization ID**: 在项目信息区域或账户设置中

## 三、添加到GitHub Secrets

### 📍 访问路径
```
https://github.com/[你的用户名]/[仓库名]/settings/secrets/actions
```

### 📝 添加三个Secrets

点击 **New repository secret** 依次添加：

#### Secret 1
```
Name: VERCEL_TOKEN
Value: [粘贴您的Vercel Token]
```

#### Secret 2
```
Name: ORG_ID
Value: [粘贴您的orgId，如 team_xxx]
```

#### Secret 3
```
Name: PROJECT_ID
Value: [粘贴您的projectId，如 prj_xxx]
```

## ⚠️ 常见错误

### ❌ Token复制错误
- **问题**: Token包含空格或换行
- **解决**: 重新复制，确保是完整的一行

### ❌ Secret名称错误  
- **问题**: 名称大小写不对或有空格
- **解决**: 必须完全匹配：`VERCEL_TOKEN`, `ORG_ID`, `PROJECT_ID`

### ❌ 找不到.vercel文件夹
- **问题**: 还没运行 `vercel link`
- **解决**: 先运行 `npx vercel link` 链接项目

### ❌ orgId格式不对
- **问题**: 可能复制了团队名称而不是ID
- **解决**: 应该是 `team_xxx` 或 `user_xxx` 格式
